Section 01

Project Brief

Createch Hobbies is a Kenya-based business that sells DIY educational kits for children. The client needed a complete e-commerce platform that could sell products online, accept mobile money payments, and be managed independently after handover.

Client Requirements

Business Requirements

  • Sell DIY kits online to Kenyan families
  • Accept M-Pesa as the primary payment method
  • Display products with age ranges and difficulty levels
  • Show a gift guide to help parents choose kits
  • Allow customers to contact via WhatsApp
  • View and manage orders from a dashboard

Operational Requirements

  • Client must be able to add and update products independently
  • Price changes must go live without developer help
  • No need to understand code to manage the store
  • Orders must be trackable and exportable
  • Website must be fast and mobile-friendly
  • Secure: customer data and API keys protected

Constraints and Context

  • Kenya market: M-Pesa is essential, card payments are secondary
  • Client is non-technical; the management interface must be simple and familiar
  • Existing WordPress/WooCommerce hosting on cPanel (Cloudoon) was already in place
  • Budget favoured using managed cloud services over self-hosted infrastructure

What Was Delivered

Customer-Facing Store

  • Homepage with video background and featured kits
  • Full product catalogue with category and difficulty filters
  • Individual product pages with image gallery
  • Gift guide filtered by age, interest, and budget
  • Shopping cart with quantity management
  • M-Pesa, Airtel Money, and card checkout
  • Order confirmation page with order reference
  • WhatsApp integration on every page

Admin Dashboard

  • PIN-protected login
  • Live order feed (paid / pending / failed)
  • Revenue and order KPI cards
  • Order table with search and status filter
  • Per-order detail view with payment receipt
  • Customer directory with order history
  • CSV export for bookkeeping

Section 03

Building the Frontend

The frontend is a Next.js 15 application using the App Router. It is deployed on Vercel and serves the customer-facing store at createch-hobbies.co.ke as well as the admin dashboard.

Application Structure

Customer Pages

  • /: Homepage (hero, featured kits, categories)
  • /shop: Full product catalogue with filters
  • /shop/[slug]: Individual product page
  • /gift-guide: Filtered gift recommendations
  • /cart: Shopping cart
  • /checkout: Payment selection and order form
  • /order-confirmation: Post-payment confirmation

Admin Pages + API Routes

  • /admin: PIN-protected dashboard
  • /admin/orders: Orders table with search
  • /admin/customers: Customer directory
  • /api/products: Products proxy (WooCommerce)
  • /api/products/[slug]: Single product proxy

Key Technical Decisions in the Frontend

Why a custom store instead of WooCommerce's built-in storefront? WooCommerce has its own shopping pages, but they are generic and difficult to customise deeply without PHP knowledge. Building the storefront in Next.js gave complete design freedom. The yellow brand colour, the custom product cards, the gift guide, the animated category filters: none of this would have been practical in standard WooCommerce themes. WooCommerce is used only as a data source via its REST API.
ISR (Incremental Static Regeneration): Product pages are pre-rendered at build time and cached on Vercel's global CDN. Every 60 seconds, Vercel silently regenerates the cache in the background. This means the store is as fast as a static website while still showing live prices and stock from WooCommerce. A customer in Mombasa and a customer in Nairobi both get served from the nearest Vercel edge node.

Product Data Strategy

The store uses a smart merge between live WooCommerce data and built-in default data. For every product field, it checks WooCommerce first. If the client has filled in the field, it uses that value. If the field is blank in WooCommerce, the built-in default is used as a fallback. This lets the client fill in product details gradually over time without anything breaking.

Section 04

Building the Backend

The backend is a Django 5 application deployed on Railway. It handles all payment processing, order storage, and provides the REST API consumed by the admin dashboard. It does not serve the storefront. That is Vercel's job.

Application Structure

Django Apps

  • orders/: Order and OrderItem models, create and retrieve API
  • mpesa/: Legacy Safaricom Daraja integration (kept as fallback)
  • ipay/: iPay Africa IPN receiver and order status updater

API Endpoints

  • POST /api/orders/: Create order, trigger payment
  • GET /api/orders/: List all orders (admin only)
  • GET /api/orders/<id>/: Single order detail
  • POST /api/orders/<id>/mark-paid/: Mark order paid (called from iPay IPN)
  • POST /api/orders/<id>/mark-failed/: Mark order failed
  • POST /api/mpesa/callback/: Legacy M-Pesa callback (kept as fallback)
  • GET /api/health/: Service health check

Why Django and not Node.js? The backend handles financial transactions and order records. Correctness and reliability matter more than raw speed. Django's ORM gives strict data validation, its migrations system ensures the database schema is always in a known state, and Python's ecosystem has mature libraries for every payment provider available in Kenya.

Section 06

WooCommerce Integration

WooCommerce is connected as a headless CMS. It acts purely as a product database. Customers never see the WooCommerce interface. The Next.js store reads product data from WooCommerce's REST API every 60 seconds and displays it in the custom design. All product fields, including categories, descriptions, age ranges, and difficulty levels, are managed directly from WooCommerce.

What "headless" means in practice: Standard WooCommerce ships with its own shopping pages that you customise. In this setup those pages are completely unused. WooCommerce is connected via its REST API, the same way a mobile app would connect to a backend. The client manages products in a familiar WordPress interface; the data flows automatically into the custom Next.js store within 60 seconds of saving.

What WooCommerce Controls

WooCommerce is the single source of truth for all product data:

  • Product prices (in KES)
  • In stock / out of stock status
  • Product images
  • Featured product flag (homepage)
  • Product category (Vehicles, Robots, etc.)
  • Age range (e.g. 8–12)
  • Difficulty level (Beginner / Intermediate / Advanced)
  • Description and learning outcomes

Step 1: Set Up Product Categories

Do this once. The store recognises these six category names exactly.

  1. In WooCommerce, go to Products → Categories
  2. Create the following categories (spelling must match exactly): Vehicles, Machines, Science, Space, Robots, Architecture
  3. Edit each product and assign it to its correct category using the Product Categories panel in the right sidebar
  4. Click Update. The product shows under the correct category within 1 minute
Adding a brand new category (e.g. "Arts and Crafts") requires a developer. The new name must first be registered in the code. Assigning products to the six existing categories above is entirely self-service.

Step 2: Add Product Attributes

  1. Open a product and go to Product Data → Attributes
  2. Add attribute Age Range, value e.g. 8–12
  3. Add attribute Difficulty; value must be exactly: Beginner, Intermediate, or Advanced
  4. Add attribute What You Learn; multiple values separated by | e.g. Mechanics | Engineering | Physics
  5. Click Save attributes then Update the product
Attribute names are case-sensitive. The store looks for Age Range, Difficulty, and What You Learn exactly as written. A typo will cause the field to fall back to the default value.

Step 3: Add a Product Description

  1. On the product edit page, fill in the Short Description text box (below the main editor)
  2. Write 1–3 sentences describing what the child will build and learn
  3. Click Update. The description appears on the store within 1 minute
Use Short Description, not the main Description. The store reads the Short Description field. The main large description box is not displayed on the store front-end.
Section 07

Payment Integration

All payments are processed through iPay Africa, a Kenyan payment gateway that handles M-Pesa, Airtel Money, and Visa/Mastercard through a single hosted payment page. Customers are redirected to iPay's secure page to complete payment, then returned to the store.

Why iPay Africa? iPay is a Kenya-focused gateway that handles M-Pesa, Airtel Money, and card payments under one integration. The hosted-page model means no card data ever touches our servers, no STK push infrastructure to maintain on our side, and a single IPN (Instant Payment Notification) endpoint handles all payment methods. Competitive Kenya-market transaction rates with local support.

M-Pesa

Customer selects M-Pesa on iPay's hosted page. iPay sends STK push to phone. Customer enters PIN to approve.

Airtel Money

Customer selects Airtel Money on iPay's hosted page. iPay sends STK push to Airtel number. Customer enters PIN.

Visa / Mastercard

Customer enters card details on iPay's PCI-compliant hosted page. No card data touches our servers.

How does the redirect work? When a customer clicks "Pay", our Next.js server builds a signed form with the order details and an MD5 hash (using the vendor ID and datakey). The browser auto-submits that form to iPay's URL. iPay processes the payment on their side, then redirects the customer back to our confirmation page and sends an IPN notification to our server.

What is an IPN? An IPN (Instant Payment Notification) is an automatic server-to-server notification that iPay sends to our /api/ipay/callback endpoint the moment a payment completes or fails. Our server receives it, verifies the status code, and immediately updates the order status in the Django database. This is why the confirmation page shows the correct status.
What if an order stays "pending" for a long time? The customer may have closed the iPay page before completing payment, or the payment was declined. The IPN will arrive and update the status automatically. If still pending after 10 minutes, the customer can try again from the checkout page. Contact iPay Africa support with the transaction reference if a payment was deducted but the order did not update.

Changes Made

  • New lib/ipay.ts: MD5 hash builder per iPay v3 spec
  • New /api/ipay/initiate route: creates Django order, returns signed iPay form fields
  • New /api/ipay/callback route: IPN handler, updates order on Django
  • Order model: ipay_txn_code replaces intasend_invoice_id

Section 08

Domain Migration

The domain createch-hobbies.co.ke was originally pointing entirely at cPanel (WordPress). It was migrated to a split configuration: the main domain now serves the Next.js store via Vercel, while WordPress was moved to a dedicated subdomain at wp.createch-hobbies.co.ke.

What is DNS? DNS (Domain Name System) works like a phone book for the internet. When someone types createch-hobbies.co.ke into their browser, DNS looks up which server that domain points to and connects them. Before the migration it pointed to the cPanel server (WordPress). After migration it points to Vercel (the Next.js store). WordPress is still on cPanel but is now reached through a subdomain with its own separate DNS entry.

Steps Completed
  1. Created subdomain wp.createch-hobbies.co.ke in cPanel → Domains
  2. Copied all WordPress files from /createch-hobbies.co.ke/ to /wp.createch-hobbies.co.ke/ via File Manager
  3. Updated wp-config.php: added WP_HOME and WP_SITEURL constants pointing to the new subdomain. This overrides the database values so WordPress knows its new location instantly, with no database edit required
  4. Verified WordPress loaded correctly at the new subdomain before touching any DNS. This is the critical safety check that ensures zero downtime
  5. Updated DNS Zone: changed the root A record from cPanel IP to Vercel IP (76.76.21.21)
  6. Added www CNAME pointing to Vercel's DNS endpoint
  7. Added custom domain in Vercel project settings
  8. Updated WOO_URL environment variable on Vercel to point to the new wp subdomain
  9. Redeployed on Vercel, confirmed live at createch-hobbies.co.ke
The SSL certificate: cPanel was already running a wildcard certificate (*.createch-hobbies.co.ke). A wildcard covers all subdomains automatically, so wp.createch-hobbies.co.ke was already protected by HTTPS the moment it was created, so no additional certificate setup was needed.

Section 09

Client Management Guide

Everything needed to manage the store day-to-day without developer involvement.

Viewing Orders

  1. Go to createch-hobbies.co.ke/admin and enter PIN: createtech2026
  2. The dashboard shows today's revenue, order count, and recent failed payments
  3. Click Orders in the sidebar for the full order history with search and filter
  4. Click any row to see full order details including the payment reference
  5. Use Export CSV to download all orders to Excel for bookkeeping
Order statuses:
  • Pending: order placed, waiting for customer to approve STK push
  • Paid: payment confirmed, ready to pack and ship
  • Failed: customer cancelled, timed out, or had insufficient funds
  • Processing: order is being prepared
  • Shipped / Delivered: updated manually by admin

Adding a New Product

  1. Log in at wp.createch-hobbies.co.ke/wp/wp-admin
  2. Go to Products → Add New and enter the product name
  3. Write 1–3 sentences in the Short Description box (not the main description)
  4. Under Product Data → General, set Regular Price in KES (numbers only)
  5. Under Product Data → Inventory, set Stock Status to In Stock
  6. Under Product Data → Attributes, add Age Range, Difficulty, and What You Learn
  7. In the right sidebar under Product Categories, assign the correct category
  8. Upload images under Product Gallery in the right sidebar (square, 800×800px+)
  9. Tick Featured if it should appear on the homepage
  10. Click Publish. Product appears on the store within 1 minute

Updating a Price or Stock Status

  1. Go to WooCommerce → Products and click the product
  2. To change price: go to Product Data → General, update Regular Price (numbers only, e.g. 2500)
  3. To change stock: go to Product Data → Inventory, set Stock Status
  4. Click Update. Changes are live on the store within 1 minute
Common Questions
A customer was charged but their order does not appear in the dashboard.
Filter orders by "Pending". The order is there but the iPay IPN has not arrived yet. Wait 5–10 minutes. If it stays pending, contact iPay Africa support with the customer's phone number and transaction reference. They can confirm whether the payment went through on their end.
A price I updated in WooCommerce is not showing on the store yet.
Wait up to 1 minute and hard-refresh the page (Ctrl+Shift+R on Windows, Cmd+Shift+R on Mac). If it still does not update after 3 minutes, confirm the price was actually saved in WooCommerce. Check that you clicked Update and the value shows correctly when you re-open the product.
Can I assign products to categories like Vehicles or Robots from WooCommerce?
Yes, fully supported. Create the six categories (Vehicles, Machines, Science, Space, Robots, Architecture) under Products → Categories if they do not exist, then assign each product. See Section 06 for the full setup steps.
Can I add a completely new category that does not currently exist?
Not without a developer. Adding a brand new category requires a small code change to register the new name. It takes less than 30 minutes. Assigning products to the six existing categories is fully self-service.
The website is down. What do I check first?
Visit status.vercel.com to check for a Vercel outage. If Vercel is fine, check the Vercel dashboard for a failed build from a recent code deployment. The WooCommerce admin at wp.createch-hobbies.co.ke is on a separate server and should remain accessible even when the main store is down.
